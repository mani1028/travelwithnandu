# Test Instructions for QA Team

## 1. Environment Setup
- Use the provided deployment URL or open public/driver-portal.html locally.
- Ensure internet access for Firebase/Google login.
- Use test accounts for Email/Google authentication.

## 2. Login & Registration
- Test both Email and Google login flows.
- Complete the multi-step registration form with valid and invalid data.

## 3. Profile & Data
- Verify profile creation, update, and data isolation by UID.
- Check booking/ticket data loads only for logged-in user.

## 4. Branding & UI
- Confirm logo appears on splash, login, and main screens.
- Test on desktop and mobile devices.

## 5. Reporting
- Log all issues using bug-report-template.md.
- Mark test case status in test-cases-driver-portal.xlsx.

Contact dev team for questions or access issues.
