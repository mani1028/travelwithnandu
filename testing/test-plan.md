# Travel With Nandu - Driver Portal Test Plan

## Objective
Ensure all features of the Driver Portal function as intended and meet requirements.

## Scope
- Splash/Login screen
- Authentication (Email, Google)
- Registration (multi-step)
- Profile management
- Booking/ticket viewing
- Branding/logo display

## Test Scenarios
1. Splash screen displays logo and progress
2. Login with valid/invalid credentials (Email, Google)
3. Registration form step navigation and validation
4. Profile creation and update
5. Booking/ticket data loads for correct user
6. Firestore security rules enforcement
7. Responsive/mobile layout
8. Error handling and user feedback
